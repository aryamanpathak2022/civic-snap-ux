## Pre-Interview (5 min)

**Facilitator:** Thank you for participating. Today we're testing a digital prototype for an app called Civic Snap — it helps citizens report civic issues like potholes, broken streetlights, and garbage problems. This is a high-fidelity digital prototype that you'll interact with on a phone/tablet. There are no wrong answers — we're testing the design, not you. Please think out loud as you go. I'll be recording this session. Is that okay?

**[Wait for consent]**

**Facilitator:** Before we start, a few quick questions:

1. What is your age range? (18-24 / 25-34 / 35-44 / 45-54 / 55+)
2. How would you rate your comfort with smartphone apps? (1-5)
3. Have you ever reported a civic issue to authorities? If yes, how?
4. Have you used any civic or government apps before?
5. What frustrates you most about reporting civic issues currently?
6. What would make you more likely to report an issue with an app?

## Task 1: Complete End-to-End Report Flow (5-7 min)

**Facilitator:** You notice water leaking from a broken pipe on your street. Report it from start to finish using the app.

**Expected Path:** Landing → Phone Login → OTP → Profile Setup → Home → Report Issue → Category → Photo → Tags → Description → Location → Review & Submit → Confirmation

**Success Criteria:**
- Completes onboarding without confusion
- Finds and taps "Report Issue" on home
- Selects correct category
- Understands photo upload step
- Uses tag system (add/remove)
- Completes location step
- Notices anonymous reporting toggle
- Sees and understands confirmation screen

**Observe:**
- Time to complete full flow
- Any hesitation points
- Whether anonymous toggle is noticed
- Reaction to auto-routing information
- Understanding of confirmation details

**Prompts (only if stuck):**
- "What would you try next?"
- "What are you looking for on this screen?"

---

## Task 2: Track & Verify a Report (5-7 min)

**Facilitator:** Check the status of your most recent report. If it shows as resolved, verify the fix.

**Expected Path:** Home → Reports tab → Select report → Tracking screen → (expand details) → Verify fix → Confirm/Dispute

**Success Criteria:**
- Navigates to Reports via bottom nav
- Identifies correct report
- Understands progress steps
- Uses "Show Details" toggle
- Reads activity history
- Understands field worker tasks
- Completes verification flow

**Observe:**
- Whether user expands details or stays on simple view
- Understanding of "Progress Steps" (vs old "Stage-gates")
- Reaction to activity history
- Understanding of dispute options
- Any confusion about karma/trust points

---

## Task 3: Browse Nearby Issues (3-5 min)

**Facilitator:** You're curious about what other issues people in your area have reported. Browse them on the map.

**Expected Path:** Home → Map View → Browse pins → Tap pin → View details → Support report

**Success Criteria:**
- Finds map view
- Understands color-coded pins
- Uses filters (if available)
- Views public report details
- Understands "Support This Report"

**Observe:**
- Map interaction comfort
- Filter usage
- Understanding of public vs private info

---

## Task 4: Profile & Karma (3-5 min)

**Facilitator:** Check your profile and see your karma/trust points. Understand what they mean.

**Expected Path:** Home → Profile tab → View karma → Read explanation → See levels

**Success Criteria:**
- Finds profile section
- Understands karma point value
- Reads karma level explanation
- Understands trust point benefits

**Observe:**
- Is karma system clear now (vs paper testing)?
- Does the level system make sense?
- Is it motivating or gimmicky?

## Post-Interview (10 min)

### Task-Specific Questions
1. Which task was the easiest? Which was the hardest? Why?
2. Was there any screen where you weren't sure what to do next?
3. Did any label, icon, or button confuse you?
4. How did you feel about the number of steps required to report an issue?
5. Did you understand what happens after you submit a report?
6. Did you notice the improvements we made since the paper version? (Show specific changes if needed)

### Open-Ended Questions
1. If you could change one thing about this app, what would it be?
2. Would you trust this app with your location data? Why or why not?
3. Would you recommend this app to a friend? Why or why not?
4. How does this compare to the paper version you saw earlier (if applicable)?
5. Any additional comments or suggestions?
